# YNAB (You Need A Budget) — Software Architecture

**Platform:** iOS · **Language:** Swift · **UI:** SwiftUI · **Pattern:** MVVM

---

## 1. Recommended Modules

| # | Module | Purpose |
|---|---|---|
| 1 | **Auth** | PIN / biometric lock screen |
| 2 | **Dashboard** | Balance summary, recent transactions, quick-add |
| 3 | **Transactions** | Full ledger — add, edit, delete income & expenses |
| 4 | **Budgets** | Monthly spending limits per category |
| 5 | **Categories** | Custom income / expense categories with icons |
| 6 | **Accounts** | Wallets (cash, bank, card) with individual balances |
| 7 | **Reports** | Charts — spending by category, income vs expense trends |
| 8 | **Recurring** | Scheduled repeating transactions (subscriptions, salary) |
| 9 | **Export** | CSV / PDF export of transaction history |
| 10 | **Settings** | Currency, theme, notification preferences, data reset |

## 3. Data Models

> All models conform to `Codable` and `Identifiable`. The `id` field maps to the Firestore **document ID** via `@DocumentID`. All user data is scoped under `users/{uid}/` in Firestore.

```
Account                              Firestore: users/{uid}/accounts/{id}
├── id: String?  (@DocumentID)       auto-generated by Firestore
├── name: String                     e.g. "BDO Savings", "Cash Wallet"
├── type: cash | bank | credit | investment
├── balance: Double
├── currency: String                 e.g. "PHP", "USD"
├── color: String                    hex for UI
└── createdAt: Date                  Firestore Timestamp

Category                             Firestore: users/{uid}/categories/{id}
├── id: String?  (@DocumentID)
├── name: String                     e.g. "Food", "Transport", "Salary"
├── icon: String                     SF Symbol name
├── color: String
└── type: income | expense

Transaction                          Firestore: users/{uid}/transactions/{id}
├── id: String?  (@DocumentID)
├── amount: Double
├── type: income | expense | transfer
├── categoryId: String               → Category document ID
├── accountId: String                → Account document ID (source)
├── toAccountId: String?             → Account document ID (destination, transfers only)
├── date: Date                       Firestore Timestamp
├── note: String?
├── isRecurring: Bool
└── recurringId: String?             → RecurringRule document ID

RecurringRule                        Firestore: users/{uid}/recurringRules/{id}
├── id: String?  (@DocumentID)
├── title: String
├── amount: Double
├── type: income | expense
├── categoryId: String
├── accountId: String
├── frequency: daily | weekly | monthly | yearly
├── startDate: Date
├── nextDueDate: Date
└── isActive: Bool

Budget                               Firestore: users/{uid}/budgets/{id}
├── id: String?  (@DocumentID)
├── categoryId: String               → Category document ID
├── limit: Double
├── period: monthly | weekly
└── createdAt: Date

BudgetProgress                       (computed locally, not persisted)
├── budget: Budget
├── spent: Double
├── remaining: Double
├── percentUsed: Double
└── isOverBudget: Bool

UserSettings                         Firestore: users/{uid}/settings/preferences
├── currency: String                 e.g. "PHP"
├── currencySymbol: String           e.g. "₱"
├── isPINEnabled: Bool
├── pin: String?                     (store hashed)
├── isBiometricEnabled: Bool
├── colorTheme: system | light | dark
└── notificationsEnabled: Bool
```

              ├── AccountListView

FirebaseService (generic Firestore CRUD helper)
  ├── .add<T: Codable>(to:document:)           → writes document
  ├── .update<T: Codable>(in:id:document:)     → updates document
  ├── .delete(from:id:)                        → deletes document
  ├── .fetch<T: Codable>(from:)                → one-shot fetch
  └── .listen<T: Codable>(to:) → AnyPublisher  → real-time listener

  ├── RecurringService   .processDueTransactions(rules:) → [Transaction]

  ├── ExportService      .generateCSV(transactions:)     → String

## 7. Screen Specifications

### LockView *(Auth)*
- Shown on cold launch if PIN or biometric is enabled
- Face ID / Touch ID prompt fires automatically
- Fallback to 6-digit PIN pad
- Three wrong attempts → 30-second cooldown

### DashboardView
- **Total Balance card** — sum across all accounts
- **This Month** — income vs expense at a glance
- **Budget alerts** — any over-limit categories surface here
- **Recent Transactions** — last 5, tapping opens full list
- **FAB** → `AddTransactionSheet`

### TransactionListView
- Grouped by date (Today, Yesterday, date headers)
- Filter chips: All · Income · Expense · Transfer
- Search by note / amount
- Swipe-to-delete with confirmation
- Pull-to-refresh triggers `RecurringService.processDueTransactions()`

### AddTransactionSheet
- Amount input with currency symbol prefix
- Type toggle: Income / Expense / Transfer
- Category picker (filtered by type)
- Account picker (+ destination account for Transfer)
- Date picker (defaults to today)
- Optional note field
- Toggle: "Make this recurring" → reveals frequency picker

### ReportsView
- Period selector: Week / Month / Year / Custom range
- **Donut chart** — category breakdown of expenses
- **Grouped bar chart** — income vs expense per sub-period
- **Line chart** — net balance trend
- Tap a chart segment → drill into filtered `TransactionListView`

### BudgetListView
- One card per budget: category icon, limit, spent, progress bar
- Color: green → yellow (75%) → red (100%+)
- Overspent budgets float to top
- FAB → `AddBudgetSheet` (category, limit, period)

### AccountListView
- Cards showing account name, type, balance
- Swipe-to-delete (blocks if account has transactions — prompt to reassign)
- Tap → filtered `TransactionListView` for that account

### RecurringListView
- List of all recurring rules with next due date
- Toggle active / paused per rule
- Swipe-to-delete

### SettingsView
- Currency selection (affects display only, does not convert)
- Theme: System / Light / Dark
- Security: enable PIN, enable biometric
- Export: CSV or PDF date-range picker → share sheet
- Danger zone: Clear all data (requires PIN confirmation)

## 8. Key Operations

| Operation | Method |
|---|---|
| Add transaction | `store.addTransaction(...)` — updates account balance |
| Edit transaction | `store.updateTransaction(...)` — reverses old, applies new |
| Delete transaction | `store.deleteTransaction(id:)` — reverses balance |
| Transfer between accounts | `store.addTransfer(amount:from:to:date:note:)` |
| Add / delete account | `store.addAccount(...)` / `store.deleteAccount(id:)` |
| Add / delete category | `store.addCategory(...)` / `store.deleteCategory(id:)` |
| Add / delete budget | `store.addBudget(...)` / `store.deleteBudget(id:)` |
| Get budget progress | `BudgetService.progress(for:transactions:)` |
| Process recurring rules | `RecurringService.processDueTransactions(rules:)` |
| Export CSV | `ExportService.generateCSV(transactions:)` |
| Export PDF | `ExportService.generatePDF(transactions:)` |

## 9. Balance Integrity Rules

```
Add Income     → account.balance += amount
Add Expense    → account.balance -= amount
Add Transfer   → source.balance -= amount  |  destination.balance += amount

Edit Transaction
  Step 1: Reverse the original transaction's balance effect
  Step 2: Apply the updated transaction's balance effect

Delete Transaction
  → Reverse the transaction's original balance effect
```

> Balances are always recomputed from the source transaction — never stored as a running total alone, preventing drift.

## 10. Persistence — Firebase

YNAB uses **Firebase** as its backend. All user data lives in **Cloud Firestore** scoped per authenticated user.

### Firestore Database Structure

```
ynab-app (Firestore root)
└── users/
    └── {uid}/
        ├── accounts/          ← Account documents
        │   ├── {accountId}
        │   └── ...
        ├── categories/        ← Category documents
        │   ├── {categoryId}
        │   └── ...
        ├── transactions/      ← Transaction documents
        │   ├── {transactionId}
        │   └── ...
        ├── recurringRules/    ← RecurringRule documents
        │   ├── {ruleId}
        │   └── ...
        ├── budgets/           ← Budget documents
        │   ├── {budgetId}
        │   └── ...
        └── settings/
            └── preferences   ← Single UserSettings document
```

### Firestore Security Rules

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users can only access their own data
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null
                         && request.auth.uid == userId;
    }
  }
}
```

### Phase 1 — Firestore + Anonymous Auth *(from the start)*

```swift
// YNABApp.swift
import Firebase

@main
struct YNABApp: App {
    init() {
        FirebaseApp.configure()
    }
}

// AuthService.swift
class AuthService: ObservableObject {
    @Published var user: User?
    
    func signInAnonymously() async throws {
        let result = try await Auth.auth().signInAnonymously()
        self.user = result.user
    }
}

// FirebaseService.swift — generic Firestore helper
class FirebaseService {
    private let db = Firestore.firestore()
    
    func userCollection(_ name: String) -> CollectionReference {
        let uid = Auth.auth().currentUser!.uid
        return db.collection("users").document(uid).collection(name)
    }
    
    func add<T: Codable>(_ item: T, to collection: String) async throws {
        try userCollection(collection).addDocument(from: item)
    }
    
    func listen<T: Codable>(to collection: String) -> some Publisher<[T], Error> {
        // Returns real-time snapshot listener as a Combine publisher
    }
}
```

### Phase 2 — Email / Apple Sign-In

```swift
// Upgrade anonymous account to permanent
func linkEmail(email: String, password: String) async throws {
    let credential = EmailAuthProvider.credential(withEmail: email, password: password)
    try await Auth.auth().currentUser?.link(with: credential)
}
```

Enables cross-device sync automatically — Firestore data is already in the cloud.

### Phase 3 — Keychain

Store PIN hash and biometric preference in Keychain (never UserDefaults).

### Phase 4 — Offline Support

Firestore has built-in offline persistence enabled by default on iOS. Users can read and write data while offline; changes sync automatically when connectivity is restored.

```swift
// Offline persistence is enabled by default, but can be configured:
let settings = FirestoreSettings()
settings.isPersistenceEnabled = true
settings.cacheSizeBytes = FirestoreCacheSizeUnlimited
Firestore.firestore().settings = settings
```

## 12. Deletion Cascade Rules

```
Delete Account
  └── blocks if transactions exist → prompt user to reassign or delete all
  └── if forced: deletes all linked transactions, reverses no balance (account gone)

Delete Category
  └── blocks if transactions exist → prompt to reassign to another category
  └── if budget exists for category: deletes budget too

Delete Budget
  └── no cascade — transactions unaffected

Delete RecurringRule
  └── past generated transactions remain
  └── future scheduled notifications cancelled
```

## 14. Build System — XcodeGen

YNAB uses [XcodeGen](https://github.com/yonaskolb/XcodeGen) to generate the Xcode project from a declarative `project.yml` file. This eliminates `.xcodeproj` merge conflicts and keeps the project configuration version-controlled as a simple YAML.

### project.yml

```yaml
name: YNAB
options:
  bundleIdPrefix: com.ynab
  deploymentTarget:
    iOS: "17.0"
  xcodeVersion: "16.0"
  generateEmptyDirectories: true

packages:
  Firebase:
    url: https://github.com/firebase/firebase-ios-sdk
    majorVersion: "11.0.0"

settings:
  base:
    SWIFT_VERSION: "6.0"
    MARKETING_VERSION: "1.0.0"
    CURRENT_PROJECT_VERSION: 1

targets:
  YNAB:
    type: application
    platform: iOS
    sources:
      - path: YNAB
        excludes:
          - "**/*.tests.swift"
    dependencies:
      - package: Firebase
        product: FirebaseAuth
      - package: Firebase
        product: FirebaseFirestore
    settings:
      base:
        INFOPLIST_FILE: YNAB/Info.plist
        PRODUCT_BUNDLE_IDENTIFIER: com.ynab.app
        CODE_SIGN_STYLE: Automatic
    info:
      path: YNAB/Info.plist
      properties:
        CFBundleName: "YNAB"
        CFBundleDisplayName: "YNAB"
        CFBundleShortVersionString: "$(MARKETING_VERSION)"
        CFBundleVersion: "$(CURRENT_PROJECT_VERSION)"
        UILaunchScreen: {}
        NSFaceIDUsageDescription: "YNAB uses Face ID to secure your financial data."

  YNABTests:
    type: bundle.unit-test
    platform: iOS
    sources:
      - path: YNABTests
    dependencies:
      - target: YNAB
    settings:
      base:
        PRODUCT_BUNDLE_IDENTIFIER: com.ynab.app.tests
```

### Usage

```bash
# Install XcodeGen (once)
brew install xcodegen

# Generate the Xcode project
xcodegen generate

# Open in Xcode
open YNAB.xcodeproj
```

> **Note:** The `YNAB.xcodeproj` directory is git-ignored. Every developer runs `xcodegen generate` after cloning to create their local project file.
